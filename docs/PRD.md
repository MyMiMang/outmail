# 产品需求文档 (PRD)

> 项目代号：**outmail / webmail**
> 文档版本：v1.0
> 最后更新：2026-05-08

## 1. 一句话定义

一个基于浏览器的 Outlook / Hotmail 邮箱**批量取件 & 验证码提取**工具，凭据由用户自己保管在浏览器，服务端只做转发不存储。

## 2. 目标用户

- **跨境电商 / 注册号运营者**：手上有几十到上千个 Outlook/Hotmail 邮箱（含裂变别名 `user+xxx@domain`），需要批量收取注册验证码、激活邮件
- **个人用户**：临时注册账号、收件不方便登录原生客户端时
- **替代方案不便者**：原生 Outlook 网页版要逐个登录，第三方收件平台不放心把 refresh_token 交出去

## 3. 用户场景

### 场景 A：注册新账号收验证码
1. 用户在某网站注册账号，验证码发到 `myacc+xxx@hotmail.com`
2. 打开本工具，从左侧列表点中对应主邮箱
3. 在「别名过滤」框输入 `xxx`
4. 点「取验证码」→ 1 秒内得到 6 位数字 → 一键复制

### 场景 B：批量收码
1. 一批账号同时收到验证码
2. 点「批量取码」→ 所有邮箱并发拉取（8 路并发）
3. 表格视图查看每个邮箱拿到的验证码 + 主题，便于对账

### 场景 C：邮件查阅
1. 想看某封邮件正文（不是验证码场景）
2. 选中邮箱 → 拉取邮件 → 点击列表中的邮件
3. HTML / 纯文本 / 原始 JSON 三种视图切换查看

## 4. 核心功能

| # | 功能 | 描述 |
|---|---|---|
| F1 | 凭据导入 | 文本框粘贴或文件上传 `email----password----client_id----refresh_token`，每行一条 |
| F2 | 凭据本地存储 | 全部存浏览器 `localStorage`，服务端零持久化 |
| F3 | 取件双通道 | Microsoft Graph 优先，失败自动回退 IMAP XOAUTH2 |
| F4 | 文件夹选择 | 全部 / 收件箱 / 垃圾箱 / 收件箱+垃圾箱 |
| F5 | 别名过滤 | 输入别名后缀，仅返回 `toRecipients` 命中的邮件 |
| F6 | 单邮箱拉信 | 拉取最近 N 封 (10/20/50/100)，列表 + 详情阅读 |
| F7 | 单邮箱取码 | 自动正则提取 4-8 位数字验证码（中英文话术） |
| F8 | 批量取码 | 全部邮箱并发跑 F7，表格输出 |
| F9 | token 测试 | 验证 `client_id + refresh_token` 是否仍有效 |
| F10 | 凭据导出 | 一键导出当前 localStorage 内邮箱为 txt |
| F11 | 邮件正文安全渲染 | HTML 内容用 `<iframe sandbox>` 隔离，禁脚本 |
| F12 | 访问控制 | 可选共享密码（`X-Access-Password` 头） |

## 5. 非功能性要求

- **性能**：单邮箱取码 < 3s（Graph 通道）；批量 50 个邮箱 < 30s
- **安全**：
  - 凭据全程不落服务端磁盘
  - 邮件 HTML 在 sandbox iframe 渲染，防 XSS
  - 公网部署强制 HTTPS（CF 或 Let's Encrypt）
  - 可选 `ACCESS_PASSWORD` 抗扫号
- **可用性**：服务端无状态，可随时 `docker compose up -d --build` 重建，浏览器内数据不丢
- **隐私**：服务端日志默认不打印凭据明细

## 6. 不在范围（Out of Scope）

- ❌ 多用户账号系统、注册登录
- ❌ 凭据云端同步 / 跨设备共享
- ❌ 主动推送（WebHook、邮件转发）
- ❌ 发邮件功能
- ❌ Gmail / 网易 / QQ 等其他邮箱服务商

## 7. 成功标准

- 单人或小团队 1 台 4C8G VPS 跑 1000 个邮箱、每天批量取码 5 次内不掉
- 服务器宕掉重启后用户浏览器刷新即恢复（数据不丢）
- 不依赖任何第三方 SaaS（除域名服务商和可选 CDN）

## 8. 演进路线

- v1.0（已上线）：核心取码 + 批量
- v1.1：定时轮询 + WebHook 推送 / 钉钉通知
- v1.2：浏览器端 localStorage 用主密码 AES 加密
- v1.3：多用户隔离（如有需求）
- v2.0：扩展 Gmail / Yahoo
